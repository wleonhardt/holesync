# plans/

Project planning + decision records for holesync.

| File | Purpose |
| --- | --- |
| `next-up.md` | Work queue (`## Queue`) and completed items (`## Done`) |
| `open-questions.md` | Unresolved questions (`## Open`) and their answers (`## Resolved`) |
| `decisions/` | One file per structural/design decision |
| `review-2026-07-03.md` | Full code review findings + prioritized improvement plan |

## Status vocabulary
`queued` · `in-progress` · `blocked` · `done` · `dropped`

## Conventions
- Queue items: one line, imperative, link to review/decision when applicable.
- Decisions: `decisions/YYYY-MM-DD-slug.md`, state the choice + why + alternatives rejected.
- Move items to `## Done` with the commit hash that closed them.
